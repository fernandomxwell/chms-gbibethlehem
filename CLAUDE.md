## Commands

```bash

composer run dev

# Run all tests
composer test
# or: php artisan test
# Run a single test: ./vendor/bin/phpunit --filter TestName

./vendor/bin/pint

# Build frontend assets for production
npm run build

# Initial setup (runs migrations, seeds, and creates menus)
php artisan app:setup

# Database
php artisan migrate
php artisan db:seed
```

## Architecture

**Monolithic Laravel 12 application** with server-side Blade rendering. No separate frontend SPA — Tailwind CSS + minimal Vite-bundled JS.

**Production runtime**: Laravel Octane + Swoole (single Docker container). The `Dockerfile` runs `php artisan octane:start --server=swoole`.

**Layer structure**:
- `app/Http/Controllers/` — thin controllers; delegate to Services
- `app/Services/` — all business logic lives here (one service per domain entity)
- `app/Http/Requests/` — input validation via Form Requests (15 classes)
- `app/Models/` — Eloquent models (User, Congregant, Activity, ServiceType, Schedule, ScheduleGroup, CongregantSchedule, CongregantActivityServiceType, Menu, CongregantActivities)
- `app/Enums/` — typed enums for Gender, Status, RecurrenceFrequency, RecurrenceDay, RecurrenceEndCondition
- `app/Libraries/Menus/` — pluggable menu system (see below)
- `app/Interfaces/` — interfaces for service abstractions

**Database drivers** (all environments): sessions, cache, and queue all use the `database` driver — no Redis required.

**Test database**: SQLite in-memory (`:memory:`) — declared in `phpunit.xml`.

## Key Packages

- **`rlanvin/php-rrule`** — recurring schedule rules (weekly services, etc.)
- **`giggsey/libphonenumber-for-php`** — international phone number validation
- **`laravel/octane`** — Swoole-powered HTTP server in production
- **`laravel/telescope`** — debugging (disabled in tests; enabled locally via `TELESCOPE_ENABLED=true`)

## Menu System

Menus are database-backed and generated by files in `app/Libraries/Menus/`. Each file must implement:
- `getActions()` — returns the menu's route/label definitions
- `getOrder()` — returns the display order

Translations live in `lang/`. The artisan command `php artisan app:create-menu {menu} {--parent=}` creates/updates a menu record. `php artisan app:setup` calls this for all menus.

To add a new menu entry: create a class in `app/Libraries/Menus/`, add its translation key to `lang/id.json`, then run `app:create-menu` or `app:setup`.

## AJAX Endpoints

Two AJAX endpoints exist for dynamic data loading (used in Blade views via JS):
- `GET /ajax/activities` → `ActivityController@ajax`
- `GET /ajax/congregants` → `CongregantController@ajax`

## Locale

Default locale is Indonesian (`id`). All user-facing strings should have entries in `lang/id.json`.
